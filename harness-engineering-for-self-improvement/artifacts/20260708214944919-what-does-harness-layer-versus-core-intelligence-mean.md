## Summary

The phrase asks where an AI system's improved capability is coming from. Core intelligence means the base model itself: its trained weights, reasoning ability, knowledge, and general skill. The harness layer means the system wrapped around the model: tools, workflow loops, memory, context management, permissions, evals, and execution environment. This section is asking: when an AI system looks more capable, is that because the model itself got smarter, or because the system around the model got better? Core intelligence = the base model's actual learned capability: reasoning, knowledge, coding ability, planning, judgment, all stored in the weights. Harness layer = the wrapper around the model: tools, workflow loops, memory, context selection, file access, tests, permissions, subagents, evals, and product/runtime design. Example: same model, weak harness: it gives one answer and forgets context. Same model, strong harness: it can edit files, run tests, remember logs, ask for permission, retry, and coordinate subtasks. The model did not change, but the total system became much more useful. Weng's question is not either/or. She is asking how much near-term progress comes from better harnesses versus better base models, and how they reinforce each other.
